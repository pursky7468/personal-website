## What it does

- Create and organize photo albums on your local machine
- Drag-and-drop album reordering
- Automatic thumbnail generation
- Full-size viewer with keyboard navigation
- Search and filter albums
- WCAG 2.1 Level AA accessibility compliance

## Tech

- **Electron** for the desktop shell
- **Vite** for fast builds and HMR during development
- **better-sqlite3** for synchronous local database access (no async ORM needed for a desktop app)

## Key Technical Decision

Used synchronous SQLite (`better-sqlite3`) instead of an async ORM. In a desktop app there's no concurrent request pressure, so synchronous access simplifies the code significantly — no `await` chains, no connection pooling.

## Lessons Learned

- Electron's main/renderer process boundary adds complexity. Keep business logic in the main process and expose it through narrow IPC channels.
- WCAG compliance in a desktop Electron app requires explicit focus management — the default browser focus ring doesn't always carry over.
